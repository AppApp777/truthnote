# TruthNote 设计规范 v1.0

---

## 一、品牌概述

**产品名**：TruthNote
**助手名**：小T
**产品定位**：选中文字，30秒告诉你哪里有问题
**品牌气质**：聪明温暖但有距离感的像素机器助手——像 R2-D2，不是 Siri
**核心视觉基因**：纯矩形、直角、黑白主色、唯一圆形是 o
**核心品牌动作**：标注——在信息上做标记，帮你区分真假，不是删除假信息，是标出来让你自己判断

## 二、Logo 与字标

### 2.1 Logo

```
来源：手绘稿矢量化
构成：大写 T 的竖横结构 + 两只像素眼睛（■ ■）
隐喻：Truth 的首字母 + 可信赖的小助手在看着信息
风格：纯矩形、零圆角、黑白、像素感
```

**Logo 尺寸规格：**

| 用途 | 尺寸 | 格式 |
|------|------|------|
| 源文件 | 512×512px | SVG |
| Chrome 商店 | 128×128px | PNG |
| 扩展管理 | 48×48px | PNG |
| 工具栏 | 32×32px | PNG |
| Favicon | 16×16px | PNG |

**安全区域**：Logo 四周留至少 10% 内容区域宽度的空白。在 512px 画板上，有效图形内容居中，四周留约 50px 内边距。

### 2.2 TruthNote 字标

每个字母尽量用直线，只保留 o 为圆形：

```

                        │

**字标参数：**

| 参数 | 值 |
|------|------|
| 字重 | Medium/Semibold |
| 字宽 | Condensed |
| 字间距 | -20px 至 -40px，紧凑排列 |

| 圆角 | 只有 o 为圆形，其余全部 0 |

**字母改造对照：**

| 字母 | 原始弯曲部位 | 改造方式 | 参考风格 |

## 三、小T 人设与文案规范

### 3.1 小T 人设

**人格**：像 R2-D2 / 挽救计划的 Rocky——聪明、温暖、可靠，但明确是机器，不是人
**表达特征**：
- 自称"小T"，不说"我"
- 句子不超过 5 个字，大多数 2-3 个字
- 用句号结尾，不用感叹号
- 不解释原因，直接给结论
- 偶尔流露关心，方式很绕：不说"我帮你"，做完加一句"看。"

**小T 不做的事**：
- 不说"我很高兴帮助你"这种拟人客套话
- 不用感叹号
- 不用 emoji（全对时例外，可能出一个 □）
- 不主动说话，只在用户触发后回应

### 3.2 文案对照表

#### 界面文案

| 场景 | 常规写法 | 小T写法 |
|------|---------|--------|
| 空状态引导 | 选中网页文字后点击核查 | 给小T一段话。 |
| 核查开始 | 正在核查... | 查。 |
| 核查进行中 | 正在搜索证据 | 查... |
| 声明提取 | 声明提取完成 | 拆好了。 |
| 搜索证据 | 正在搜索证据 | 找... |
| 交叉验证 | 正在交叉验证 | 对一对。 |
| 生成报告 | 正在生成报告 | 写。 |
| 结果出炉 | 核查完成 | N条。看。 |
| 全部为真 | 所有声明均有依据 | 都对。 |
| 全部为假 | 所有声明均不属实 | 都不对。 |
| 混合结果 | 部分声明有问题 | 看看。 |

## 七、小T 表情系统

Logo 的两只眼睛 ■ ■ 就是小T 的全部表达器官。没有嘴，没有身体，只有两颗方块。

| 状态 | 左眼 | 右眼 | 语境 | 动画 |
|------|------|------|------|------|
| 默认/待机 | ■ | ■ | 空状态，等用户 | 微呼吸：scale 0.95↔1.0，周期 2s |
| 等待输入 | ■ | ■ | 同上，略有浮动 | 上下浮动 1px，周期 1.5s |
| 正在读 | ■_ | _■ | 声明提取阶段 | 两眼交替闪，0.3s 循环 |
| 正在扫描 | ► | ■ | 搜索证据阶段 | 左眼方向性，右眼稳定 |
| 对比 | ◄ | ► | 交叉验证阶段 | 两眼交替左右看，0.5s |
| 完成 | ■ | ■ | 结果已出 | 从扫描态回到默认，眨一下 0.15s |
| 发现问题 | ■ | ■ | 有"不对"的结果 | 快闪：两眼同时缩放一次 |
| 出错 | _ | ■ | 服务异常 | 左眼变短横线，表示无奈 |
| 开心 | ■ | ■ | 全都查到了 | 右眼略偏下 1px，极克制 |

### 9.2 悬浮触点（选中文字后出现）

触发：用户选中文字后 0.3s 出现
位置：选区右下角偏移 4px
大小：约 56×28px
样式：黑底白字，0圆角

```
被选中的文字文字文字文字文字文字
        文字文字 ── ■■ 查 ─┘
网页文字文字文字文字
```

- 3秒不点 → 自动淡出
- 选区取消 → 立刻消失
- 淡入 0.2s，淡出 0.15s

### 9.3 悬浮结果卡（点击触点后展开）

宽度：320px
最大高度：400px（超出滚动）
背景：#FFFFFF，1px #000000 边框，4px 投影
圆角：0

**定位算法优先级：**
1. 选区下方（最常见）
2. 选区上方（下方空间不够）
3. 选区右侧（上下都不行）
4. 侧边栏模式（所有方向都紧张 / 代码页面）

展开动画：从触点大小 scale 展开，200ms

```
    ┌──────────────────────────────┐

### 9.4 侧边栏模式（代码页/窄屏）

触发条件（满足任一）：
- 页面宽度 < 768px
- 视口内检测到 `<code>` / `<pre>` 标签
- 悬浮卡片碰撞检测所有方向都失败
- 用户设置中勾选"总是用侧边栏"

## 十、动画规范

### 10.1 通用原则

- **简短**：所有动画 150-300ms，不超过 500ms
- **克制**：小T的动画是像素级的偏移，不是流畅的曲线运动
- **功能性**：每个动画都有信息含义，不为装饰

### 10.2 关键动画

| 动画 | 时长 | 缓动 | 描述 |
|------|------|------|------|
| 触点出现 | 200ms | easeOut | 淡入 + scale 0.8→1.0 |
| 悬浮卡展开 | 200ms | easeOut | 从触点位置 scale 展开 |
| 侧边栏滑入 | 300ms | easeOut | 从右侧滑入 |
| 小T呼吸 | 2000ms | easeInOut | scale 0.95↔1.0，无限循环 |
| 小T眨眼 | 150ms | easeOut | 眼睛 scale 1→0→1，快闪 |
| 小T扫描 | 500ms | linear | 两眼 ■_ → _■ → ■■ 交替 |
| 声明卡片出现 | 300ms | easeOut | stagger 延迟，每张卡间隔 80ms |
| 判定标签出现 | 200ms | easeOut | 从 scale 0→1 弹出 |
| 标注底色淡入 | 300ms | easeOut | 透明度 0→4% |

### 10.3 状态转换流程

```
用户选中文字
       │
       ▼
  ■■ 查（触点淡入 200ms）
       │
  用户点击触点
       │
       ▼
  小T眼睛 → ►■（查 150ms）
  触点 expand → 悬浮卡/侧边栏（200ms）
       │
       ▼
  步骤逐条出现（每条 stagger 300ms）
       │
  核查完毕
       │
       ▼
  小T眼睛 → ■■（眨一下 150ms）
  卡片内容淡入最终结果（300ms）
       │
  用户点击卡片外 / Esc
       │
       ▼
  卡片收起 → 消失（200ms）
  原文标注保留（持续显示）
```
